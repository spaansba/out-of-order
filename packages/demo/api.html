API reference

core and reveal, in full

Two packages. @focuspocus/core is the analyzer and its types, and @focuspocus/reveal is the overlay. The test-runner matcher is in progress and not documented here yet.

core · analyzeTabOrder

analyzeTabOrder(root?, options?, customRules?) → TabOrderResult

Computes the tab sequence for root and grades it against the enabled rules. It is browser-only, because tabbable uses real CSS layout for visibility and several rules read bounding rects.

root: ParentNode = document
The subtree to analyze. A Document analyzes the whole page, exactly like tabbing through it.
options: AnalyzeOptions = {}
Per-rule severity overrides. See below.
customRules: Rule[] = []
Extra rules run alongside the built-ins. See custom rules.

core · result types

What analyzeTabOrder hands back.

interface TabOrderResult {
  // True when no enabled rule produced an `error`. Warnings don't flip it.
  valid: boolean;
  // Elements in the exact order tabbing reaches them.
  sequence: SequenceEntry[];
  // All violations found, across every enabled rule (both severities).
  violations: Violation[];
}

interface SequenceEntry {
  element: Element;
  selector: string;
  orderIndex: number; // zero-based position in the tab sequence
  tabIndex: number; // resolved tabindex (0 if unset but focusable)
  rect: DOMRect; // bounding rect at analysis time (real layout)
}

interface Violation {
  rule: RuleId | (string & {}); // built-in id, or a custom rule's id
  severity: Severity; // "error" | "warning"
  message: string;
  docs?: string; // spec link for the rule
  element: Element; // the offending element
  selector: string; // a CSS-ish path, for messages and logs
  orderIndex?: number; // present when the element is a tab stop
  relatedElements?: Element[]; // same root cause, ringed but not re-counted
}

relatedElements is how one missing fix avoids becoming N separate findings. The analyzer reports a single violation, and the overlay rings every related element too.

core · options & severity

type Severity = "error" | "warning";
type RuleSetting = Severity | "off";

interface AnalyzeOptions {
  // Per-rule overrides. Omit a rule to keep its default; otherwise
  //   "error" | "warning" -> enable, graded at that severity
  //   "off"               -> disable the rule
  rules?: Partial<Record<RuleId, RuleSetting>>;
}

// The built-in defaults, as a Record<RuleId, Severity>.
const DEFAULT_SEVERITY: Record<RuleId, Severity>;

RuleId is the union of built-in rule ids, so overrides autocomplete. The full list is on the rules page.

core · helpers & exports

formatViolations(violations: Violation[]) => string
Renders violations into a readable multi-line block for test messages and logs. Each finding gets its severity, rule id, tab position, message, and docs link.
isInteractive(el: Element) => boolean
Whether an element is interactive (a native control or an interactive ARIA role). The same predicate several rules use.
selectorFor(el: Element) => string
A short, CSS-ish path to an element, for messages and logs.
ALL_RULES
The built-in rule definitions, keyed by id.
RULE_DOCS
The spec link for each rule id.
OVERLAY_CLASS_PREFIX
The class prefix the overlay marks its own nodes with, so the analyzer can ignore them instead of mistaking them for page content.

core · custom rules

A rule is a pure function over the computed sequence. Pass your own alongside the built-ins and they flow into the result and the overlay like any other. 

interface Rule {
  id: string; // surfaced on every Violation it produces
  docs: string; // spec link the rule is grounded in
  defaultSeverity: Severity;
  run: RuleRun;
}

type RuleRun = (sequence: SequenceEntry[], ctx: RuleContext) => Finding[];

interface RuleContext {
  container: Element; // the analyzed root (look beyond the sequence)
  inSequence: Set<Element>; // fast "is this a tab stop?" test
}

interface Finding {
  message: string;
  target: SequenceEntry | Element; // a SequenceEntry when it's a tab stop
  relatedElements?: Element[];
}

analyzeTabOrder stamps the severity on (from defaultSeverity or the caller's override), which is why a Finding is a Violation without one. 

import { analyzeTabOrder } from "@focuspocus/core";
import type { Rule } from "@focuspocus/core";

// Flag links that open a new tab without rel="noopener".
const noUnsafeBlank: Rule = {
  id: "no-unsafe-blank",
  docs: "https://html.spec.whatwg.org/multipage/links.html#link-type-noopener",
  defaultSeverity: "warning",
  run: (sequence) =>
    sequence
      .filter(
        (entry) =>
          entry.element.matches('a[target="_blank"]') &&
          !(entry.element.getAttribute("rel") ?? "").includes("noopener"),
      )
      .map((entry) => ({
        message: 'Link opens a new tab without rel="noopener".',
        target: entry,
      })),
};

analyzeTabOrder(document.body, {}, [noUnsafeBlank]);

reveal · reveal

reveal(options?: RevealOptions) → RevealHandle

Mounts the overlay layer, draws the first analysis, and starts observing. It re-analyzes itself on DOM mutation, so most pages need nothing past the call. 

interface RevealOptions {
  root?: ParentNode; // subtree to analyze; defaults to document
  analyze?: AnalyzeOptions; // forwarded rule toggles
  rules?: Rule[]; // extra custom rules, run with the built-ins
  exclude?: Element | null; // a subtree to skip (another overlay owns it)
  motion?: "auto" | "on" | "off"; // animation; defaults to "auto"
}

motion: "auto" follows the user's prefers-reduced-motion setting, live.

reveal · RevealHandle

What reveal() returns.

refresh(): void
Re-run the analysis and redraw. Call after the DOM changes if you have disabled or out-paced the auto-observer.
reposition(): void
Re-read every marker's rect and redraw, without re-analyzing.
visible: boolean (readonly)
Whether the overlay is currently shown.
setVisible(visible: boolean): void
Show or hide the whole overlay, including badges, lines, and rings.
toggle(): void
Flip between shown and hidden.
destroy(): void
Remove the overlay layer, observers, and listeners.
result: TabOrderResult | null
The latest analysis result, or null before the first draw.
