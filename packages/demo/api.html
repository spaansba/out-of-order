API reference

core and reveal, in full

Two packages. @focuspocus/core is the analyzer and its types, and @focuspocus/reveal is the overlay. Each entry keeps a function together with the types it takes and hands back. The test-runner matcher is in progress and not documented here yet.

audit()function

audit(root?, options?, customRules?) → AuditResult

Computes the tab sequence for root and grades it against the enabled rules. It runs in the browser only, because tabbable uses real CSS layout for visibility and several rules read bounding rects.

Parameters

Parameter Type Description
root ParentNode Optional, defaults to document. The subtree to analyze. A Document analyzes the whole page, exactly like tabbing through it.
options AuditOptions Optional. Per-rule severity overrides. See Options & severity.
customRules Rule[] Optional. Extra rules that run alongside the built-ins. See custom rules.

Returns → AuditResulttype

Holds the computed tab sequence, every violation found, and the overall verdict.

Property Type Description
valid boolean True when no enabled rule produced an error. Warnings are advisory and never flip it, so read violations when you want to surface those too.
sequence SequenceEntry[] Every tab stop, in the exact order the Tab key reaches it.
violations Violation[] Every finding, across all enabled rules, at both severities.

SequenceEntrytype

One entry per tab stop in AuditResult.sequence.

Property Type Description
element Element The focusable node at this stop.
selector string A short, CSS-ish path to the element, for messages and logs.
orderIndex number Its zero-based position in the tab sequence.
tabIndex number The resolved tabindex, 0 when the element is focusable without one.
rect DOMRect The element's bounding rect at analysis time, from real browser layout. This is why several rules need a real browser.

Violationtype

One entry per finding in AuditResult.violations.

Property Type Description
rule string The id of the rule that fired, a built-in id or a custom rule's own. Stable, so you can key or filter on it.
severity "error" | "warning" How serious the finding is, after any per-call override.
message string A plain-language account of what is wrong and how to fix it.
docs? string A spec link (WCAG, WAI-ARIA, or ARIA APG) for the rule.
element Element The offending element.
selector string A CSS-ish path to that element, for messages and logs.
orderIndex? number Its position in the tab sequence, present only when it is a stop.
relatedElements? Element[] Other elements that share this finding's root cause. Ringed alongside element but not reported as separate violations, so one missing fix does not become N findings. The overlay rings every one, and formatViolations lists their selectors.

Options & severity

The severity values and the overrides object shared by audit() and reveal().

Severity & RuleSettingtypes

The grades a rule reports at, and what an override can set it to.

Type Definition Description
Severity "error" | "warning" The two grades a rule can report at.
RuleSetting Severity | "off" What you set a rule to in an override. The value "off" disables the rule.

AuditOptionstype

The overrides object, with a single field.

Property Type Description
rules? Partial<Record<RuleId, RuleSetting>> Per-rule overrides. Omit a rule to keep its default. Set "error" or "warning" to enable it at that grade, or "off" to disable it.

RuleId is the union of the built-in rule ids, so overrides autocomplete. The full list is on the rules page.

Helpers & exports

Three helper functions and one constant.

Export Signature Description
formatViolations (violations: Violation[]) => string Renders violations into a readable multi-line block for test messages and logs. Each finding gets its severity, rule id, tab position, message, docs link, and the selectors of any related elements.
isInteractive (el: Element) => boolean Returns whether an element is interactive, meaning a native control or an element with an interactive ARIA role.
selectorFor (el: Element) => string Returns a short, CSS-ish path to an element, useful for messages and logs.

Custom rules

A rule is a pure function over the computed sequence. Pass your own alongside the built-ins and they flow into the result and the overlay like any other.

Ruletype

One custom rule.

Property Type Description
id string Surfaced on every Violation the rule produces.
docs string The spec link the rule is grounded in.
defaultSeverity Severity The grade its findings get unless the caller overrides it.
run (sequence: SequenceEntry[], ctx: RuleContext) => Finding[] The rule itself, run over the sequence.

RuleContexttype

Handed to a rule's run as its second argument.

Property Type Description
container Element The analyzed root, so a rule can look beyond the tab sequence.
inSequence Set<Element> A fast "is this element a tab stop?" test.

Findingtype

Returned by a rule's run, one per issue.

Property Type Description
message string What is wrong and how to fix it.
target SequenceEntry | Element The element the finding is about, a SequenceEntry when it is a tab stop.
relatedElements? Element[] Other elements sharing the same root cause.

audit() stamps the severity onto each finding, taken from defaultSeverity or the caller's override. That is why a Finding is a Violation without a severity. 

import { audit } from "@focuspocus/core";
import type { Rule } from "@focuspocus/core";

// Flag links that open a new tab without rel="noopener".
const noUnsafeBlank: Rule = {
  id: "no-unsafe-blank",
  docs: "https://html.spec.whatwg.org/multipage/links.html#link-type-noopener",
  defaultSeverity: "warning",
  run: (sequence) =>
    sequence
      .filter(
        (entry) =>
          entry.element.matches('a[target="_blank"]') &&
          !(entry.element.getAttribute("rel") ?? "").includes("noopener"),
      )
      .map((entry) => ({
        message: 'Link opens a new tab without rel="noopener".',
        target: entry,
      })),
};

audit(document.body, {}, [noUnsafeBlank]);

reveal()function

reveal(options?) → RevealHandle

Mounts the overlay layer, draws the first analysis, and starts observing. It re-analyzes itself on DOM mutation, so most pages need nothing past the call.

@focuspocus/reveal re-exports the whole core API, so overlay users get the headless path from this one package without installing core separately.

Parameters → RevealOptionstype

Every field is optional.

Property Type Description
root? ParentNode The subtree to analyze. Defaults to document.
audit? AuditOptions Rule overrides, forwarded to audit().
rules? Rule[] Extra custom rules, run alongside the built-ins on every analysis.
motion? "auto" | "on" | "off" Animation. Defaults to "auto", which follows the user's prefers-reduced-motion setting, live.
peekKey? "Alt" | "Control" | "Shift" | "Meta" Tap this modifier to peek through the overlay at the page beneath without hiding it. Tap it again to restore. Defaults to "Alt".

Returns → RevealHandletype

For driving the overlay after the mount.

Member Type Description
visible boolean Read-only. Reports whether the overlay is currently visible.
setVisible (visible: boolean) => void Shows or hides the whole overlay, including its badges, lines, and rings.
toggle () => void Flips the overlay between shown and hidden.
destroy () => void Removes the overlay layer, observers, and listeners.
result AuditResult | null The latest analysis result, or null before the first draw.
