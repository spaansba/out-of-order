Getting started

From install to overlay in a minute

Drop the analyzer into your own page, or point the CLI at any URL. Either way it runs in a real browser, which is the whole point. The test-runner matcher is a separate package that is still in progress, so it is not covered here yet.

See it before you install

The playground is every focus and keyboard bug the analyzer catches, each one wired to break then get fixed live, with the overlay tracing the real tab path.

Which package, and which mode

One analyzer, three ways to run it. See it on a live page with the overlay, check it headless in a test or CI, or scan any URL from the terminal.

See it, the overlay:

npm i @out-of-order/trace

import { trace } from "@out-of-order/trace";
trace();

Check it, headless:

npm i @out-of-order/core

import { audit } from "@out-of-order/core";
const result = audit();

Scan it, any URL from the terminal:

npx @out-of-order/cli https://example.com           # headless, prints findings
npx @out-of-order/cli https://example.com --overlay # opens a browser with the overlay

Add the overlay to your page

@out-of-order/trace is the shipped, framework-agnostic overlay. Import it and call trace(), and you have numbered tab stops, the path between them, and every finding ringed in place. 

import { trace } from "@out-of-order/trace";

// No options, so it analyzes the whole document.
const overlay = trace();

// Later, tear it down. This removes the layer, observers, and listeners.
overlay.destroy();

The overlay re-analyzes itself on every DOM mutation, so most pages need nothing beyond the mount. You can scope it to a subtree, hide and show it, or feed it rule overrides through trace(options). See the trace API.

The packages target npm as @out-of-order/core and @out-of-order/trace. Until they are published, consume them from the workspace (or a git install) as the demo does.

Only need the verdict, not the picture? Skip the overlay and call audit() directly, covered just below.

Analyze without the overlay

The overlay is optional. @out-of-order/core gives you the verdict as plain data, which is what you would assert in a test or feed into your own UI.

Want to see these findings on the page instead of asserting them? Mount the overlay with trace(), covered just above. 

import { audit } from "@out-of-order/core";

const result = audit();

result.valid; // boolean, true when there are no error-severity violations
result.sequence; // elements in tab order, each with tabIndex + rect
result.violations; // one entry per element: { element, selector, orderIndex, issues }

if (!result.valid) {
  // Pass `format` for a ready-to-print string instead of the structured result.
  console.log(audit(document.body, { format: "text" }).violations);
}

valid only goes false on an error. Warnings are advisory and never flip it, so they will not fail a build gate unless you promote them. See concepts for what that distinction means, and the core API for every field.

Turn rules up, down, or off

Every rule runs at a default severity. You can override any of them per call. Re-grade a noisy one to a warning, promote another to an error, or switch one off entirely. 

audit(document.body, {
  rules: {
    "visual-order-mismatch": "warning", // re-grade
    "redundant-tabindex": "error", // promote
    "prefer-native-element": "off", // disable
  },
});

// The same overrides reach the overlay through its `audit` option.
trace({ audit: { rules: { "prefer-native-element": "off" } } });

The full list, with what each rule catches, is on the rules page.

Approve a finding in place

Some findings are not bugs: a known false positive, or a deliberate choice you have already weighed. Approve one by putting data-ooo-ignore on the flagged element. 

<!-- Approve every finding on this element -->
<button data-ooo-ignore>…</button>

<!-- Approve only the named rules, leave the rest live -->
<div data-ooo-ignore="hidden-while-focusable visual-order-mismatch">…</div>

It is element-scoped, so it approves findings on that element only and is never inherited by its descendants.
